### 1\. Введение

Наименование программы: Neo Ping-Pong

Условное обозначение темы разработки: NeoPP-1.0

Документы-основания: Лабораторная работа №3 по дисциплине "Документирование программного обеспечения", 2025 г.

### 2\. Назначение и область применения

Назначение программы: обучающая 2D-игра в стиле классического пинг-понга с расширенным функционалом на базе библиотеки Pygame.

Область применения: учебные цели, демонстрация принципов разработки игр на Python, изучение обработки событий и физики объектов.

### 3\. Технические характеристики

#### 3\.1. Постановка задачи на разработку программы

Разработать игру на Python с использованием библиотеки Pygame, в которой два игрока управляют ракетками, отбивая мяч. При соприкосновении ракетки с мячом на поле появляются мины, взрывающиеся при контакте с мячом и ускоряющие его.

Математические методы: применение векторной алгебры для расчета траекторий движения, обнаружение столкновений (collision detection) с использованием прямоугольных областей (AABB).

Ограничения: используется 2D-графика, фиксированная частота обновления экрана (120 FPS).

#### 3\.2. Описание алгоритма и функционирования программы

Основной алгоритм состоит из главного игрового цикла, который обновляет состояние всех игровых объектов и отображает их на экране. Программа содержит четыре основных класса: Screen (экран и отображение), Ball (мяч с физикой), Player (ракетки игроков), Mines (взрывные объекты).

Взаимодействия: программа использует встроенные функции Pygame для обработки графики и событий клавиатуры. Возможные взаимодействия с другими модулями ограничиваются стандартной библиотекой Python.

#### 3\.3. Организация входных и выходных данных

Входные данные: события клавиатуры (W/S для игрока 1, стрелки для игрока 2), сигналы закрытия окна.

Выходные данные: графическое представление игры на экране (позиции объектов, текущие счета игроков), частота обновления 120 FPS.

#### 3\.4. Выбор состава технических и программных средств

Программные средства: Python 3.x, библиотека Pygame 2.x для создания графики и обработки событий.

Технические средства: персональный компьютер с минимальными требованиями (процессор x86/x64, ОЗУ ≥256 МБ, видеокарта с поддержкой DirectX 9.0+).

Носители данных: исходный код хранится на локальном компьютере; дополнительные ресурсы (изображения, звуки) находятся в папке assets/.

### 4\. Ожидаемые технико-экономические показатели

Производительность: программа обеспечивает стабильную частоту кадров 120 FPS при разрешении экрана 1268×850 пикселей на современных персональных компьютерах.

Потребление ресурсов: объем исходного кода \~400 строк Python, размер исполняемого пакета с зависимостями \~50-70 МБ.
