## 1\. Введение

### 1\.1. Наименование программы

Программа **«Neo Ping-Pong»** -- это 2D-игра жанра классического пинг-понга с механикой взрывающихся мин. Программа реализована на языке Python с использованием библиотеки Pygame и предназначена в учебных целях для демонстрации принципов объектно-ориентированного программирования, физики игровых объектов и управления событиями.

### 1\.2. Характеристика области применения

Программа Neo Ping‑Pong предназначена для развлекательного использования в виде 2D‑игры жанра “пинг‑понг” для двух игроков на одном компьютере. Программа может использоваться для демонстрации результата выполненной лабораторной работы (учебного проекта) по дисциплине, связанной с программированием на Python.

### 1\.3. Характеристика объекта применения

Объектом применения программы являются пользователи программы -- два игрока, управляющие ракетками с клавиатуры, и/или разработчик при демонстрации программы преподавателю.

Программа предполагает запуск на персональных компьютерах и ноутбуках с ОС Windows, оборудованных необходимыми вычислительными ресурсами.

## 3\. Назначение разработки

### 3\.1. Функциональное назначение программы

Функциональное назначение программы Neo Ping-Pong заключается в:

1. **Реализация игровых механик пинг-понга** -- программа обеспечивает базовую функциональность классической игры пинг-понг, включая управление двумя ракетками (игроками), движение мяча, обработку коллизий и подсчет очков.

2. **Расширенная механика с минами** -- добавление уникальной механики взрывающихся мин, которые появляются при отбивании мяча ракеткой и взрываются при контакте с мячом, изменяя его скорость и направление.

3. **Визуализация игрового процесса** -- отображение игровых объектов (ракеток, мяча, мин) с использованием спрайтов в стилистике «неона», вывод текущих очков игроков на экран.

4. **Обработка пользовательского ввода** -- программа обрабатывает нажатия клавиш для управления ракетками обоих игроков в реальном времени.

5. **Управление игровым состоянием** -- программа поддерживает основной игровой цикл с фиксированной частотой кадров (120 FPS).

### 3\.2. Эксплуатационное назначение программы

Программа Neo Ping‑Pong предназначена для эксплуатации конечными пользователями в режиме локальной игры «два игрока за одним компьютером» с управлением с клавиатуры. Программа рассчитана на запуск на персональном компьютере под управлением ОС семейства Windows. Эксплуатация программы не предполагает администрирования, настройки сетевого взаимодействия и работы с персональными данными. Завершение работы программы выполняется пользователем способом закрытия окна приложения.

## 4\. Требования к программе

### 4\.1. Требования к функциональным характеристикам

#### 4\.1.1 Управление игроками и ракетками

-  Программа должна обеспечивать управление двумя ракетками (красная для Player 1, синяя для Player 2)

-  Player 1 управляется клавишами W (вверх) и S (вниз)

-  Player 2 управляется клавишами ↑ (вверх) и ↓ (вниз)

-  Ракетки должны иметь ограничение движения в пределах игрового поля

-  Размер ракетки: ширина 20 пикселей, высота 200 пикселей

#### 4\.1.2 Управление мячом

-  Мяч должен отскакивать от верхней и нижней границ игрового поля

-  При выходе мяча за левую границу (x \< 0) Player 2 получает 1 очко

-  При выходе мяча за правую границу (x > WIDTH) Player 1 получает 1 очко

-  Скорость мяча должна быть переменной и зависеть от взаимодействия с минами

-  Начальное положение мяча: центр экрана

#### 4\.1.3 Механика мин

-  Мины должны появляться в случайной позиции на экране после каждого отбивания мяча ракеткой

-  При столкновении мяча с миной она должна исчезать

-  При взрыве мины мяч должен ускоряться с коэффициентом от 1.25 до 2.5

-  Мины не должны взаимодействовать с ракетками

-  Размер мины: 30x30 пикселей

-  Максимальное количество мин на экране ограничено только взаимодействиями в рамках игровой сессии

#### 4\.1.4 Подсчет и отображение очков

-  Программа должна вести счет очков для обоих игроков. Текущий счет должен отображаться на экране в верхней части игрового поля

-  Счет Player 1 (красный) отображается слева, Player 2 (синий) -- справа

-  При выходе мяча за границу поля соответствующему игроку добавляется 1 очко

-  После набора очка мяч возвращается в центр экрана, а список мин очищается

#### 4\.1.5 Визуализация

-  Экран должен отображать фоновое изображение в стиле неона

-  Размер окна: 1268x850 пикселей

-  Частота кадров: 120 FPS

### 4\.2. Требования к надежности

Программа должна корректно завершаться при нажатии кнопки закрытия окна и устойчиво работать при быстрых последовательных нажатиях клавиш.

## 8\. Порядок контроля и приемки

## 8\. Порядок контроля и приемки

### 8\.1 Вид испытаний

Программа должна пройти функциональное тестирование с целью проверки отсутствия критических ошибок при выполнении основных операций:

-  Запуск программы и инициализация игровых объектов

-  Управление ракетками обоими игроками (W/S и ↑/↓)

-  Движение и отскок мяча от границ экрана

-  Столкновение мяча с ракетками

-  Появление и взрыв мин при контакте с мячом

-  Подсчет и отображение очков

-  Корректное завершение программы при закрытии окна

### 8\.2 Критерии приемки

Программа считается принятой при выполнении следующих условий:

1. Программа запускается без ошибок при выполнении команды `python main.py`

2. Все основные функции игры работают в соответствии с описанием: управление, движение объектов, обработка коллизий

3. Во время тестирования отсутствуют критические ошибки (crashes, зависания, потеря данных)

4. Программа корректно завершается при закрытии окна

### 8\.3 Процедура приемки

1. Запуск программы и проверка основных механик в течение нескольких минут игры

2. Фиксирование найденных ошибок (если таковые имеются)

3. При отсутствии критических ошибок программа принимается в эксплуатацию

4. Если обнаружены ошибки, программа возвращается на доработку

## 9\. Приложения

### Приложение А: Словарь терминов

| Термин       | Определение                                                                                       |
|--------------|---------------------------------------------------------------------------------------------------|
| **Спрайт**   | Графический объект (изображение), используемый для представления персонажей, предметов в игре     |
| **Фрейм**    | Один кадр (frame) видеопоследовательности, обновляется с частотой FPS                             |
| **FPS**      | Frames Per Second -- количество кадров в секунду                                                  |
| **Коллизия** | Столкновение двух игровых объектов, при котором выполняется проверка пересечения их границ (rect) |
| **Ракетка**  | Игровой объект, управляемый игроком для отбивания мяча                                            |
| **Мяч**      | Летающий игровой объект, отскакивающий от границ и ракеток                                        |
| **Мина**     | Взрывающийся объект, появляющийся при отбивании мяча, ускоряющий его при столкновении             |
| **Очки**     | Единица счета, начисляемая при выходе мяча за границу противоположной стороны                     |
