## 1\. Назначение программы

### 1\.1 Назначение

Программа NeoPong является учебной 2D-игрой, реализованной на языке Python с использованием библиотеки Pygame. Программа предназначена для обучения основам разработки игр, включая работу с графикой, физикой объектов, обработкой пользовательского ввода и реализацией игровых механик.

### 1\.2 Возможности программы

-  Воспроизведение игровой сцены в реальном времени с частотой 120 FPS

-  Управление двумя игроками с независимой системой управления

-  Реализация физики движения мяча с отскоком от границ и ракеток

-  Система мин, которые появляются при соприкосновении мяча с ракетками

-  Увеличение скорости мяча при взаимодействии с минами

-  Подсчет и отображение счета обоих игроков

-  Неоновое оформление интерфейса с использованием графических ресурсов

### 1\.3 Основные характеристики

-  **Разрешение экрана**: 1268 × 850 пиксел

-  **Частота обновления кадров**: 120 FPS

-  **Язык программирования**: Python 3

-  **Используемая библиотека**: Pygame

-  **Размер мяча**: 50 × 50 пиксел

-  **Размер ракетки**: 20 × 200 пиксел

-  **Размер мины**: 30 × 30 пиксел

-  **Базовая скорость мяча**: 5 пиксел/кадр

-  **Скорость перемещения ракетки**: 10 пиксел/кадр

### 1\.4 Ограничения, накладываемые на область применения программы

-  Программа используется исключительно в образовательных целях

-  Поддерживаемая платформа: Windows, Linux, macOS (с установленными зависимостями)

-  Требует наличие графического интерфейса операционной системы

-  Игра предназначена для локального запуска (без сетевого взаимодействия)

-  Рекомендуется использование для обучения студентов основам программирования на Python

## 2\. Условия применения (требования к необходимым для данной программы техническим средствам, и другим программам, общие характеристики входной и выходной информации, а также требования и условия организационного, технического и технологического характера)

-  **Процессор**: Intel/AMD x86 или совместимый процессор

-  **Оперативная память**: минимум 512 MB

-  **Видеокарта**: встроенная или дискретная графика с поддержкой OpenGL или DirectX

## 3\. Описание задачи (определения задачи и методы ее решения)

### Определение задачи

Задача программы заключается в реализации игровой механики, аналогичной классической игре Pong, с дополнительным элементом – минами. Программа должна:

1. Инициализировать игровое поле и все игровые объекты

2. Обрабатывать ввод пользователя (движение ракеток)

3. Обновлять физику мяча (движение, отскоки)

4. Обнаруживать коллизии между объектами

5. Создавать мины при соприкосновении мяча с ракетками

6. Изменять скорость мяча при взаимодействии с минами

7. Подсчитывать очки игроков

8. Отображать состояние игры на экране

### Методы решения

**Система управления объектами**: Используется объектно-ориентированный подход с классами для каждого типа игровых объектов (мяч, ракетки, мины, экран).

**Физическая система**: Реализована простая 2D-физика с использованием координат, векторов скорости и обнаружением прямоугольных коллизий (AABB – Axis-Aligned Bounding Box).

**Цикл игры**: Главный цикл выполняется 120 раз в секунду, на каждой итерации:

-  Обрабатываются события клавиатуры

-  Обновляются позиции всех объектов

-  Проверяются коллизии

-  Отрисовываются объекты на экран

**Обнаружение коллизий**: Используется метод прямоугольных коллизий (colliderect) из библиотеки Pygame для определения пересечений объектов.

**Случайность**: Для создания различных траекторий мяча используется генератор случайных чисел (модуль random).

## 4\. Входные данные и выходные данные

### 4\.1 Характер, организация и предварительная подготовка входных данных

**Входные данные** – это события клавиатуры, которые пользователь генерирует во время игры:

-  Для первого игрока (красная ракетка): клавиши **W** (вверх) и **S** (вниз)

-  Для второго игрока (синяя ракетка): клавиши **↑** (вверх) и **↓** (вниз)

Предварительной подготовки входных данных не требуется – программа автоматически обрабатывает события из системной очереди Pygame.

### 4\.2 Формат, описание и способ кодирования входных данных

Входные данные представлены системными событиями Pygame типа pygame.KEYDOWN и pygame.KEYUP:

-  **Формат**: События операционной системы, преобразованные в события Pygame

-  **Коды клавиш**:

   -  pygame.K_w – буква W

   -  pygame.K_s – буква S

   -  pygame.K_UP – стрелка вверх

   -  pygame.K_DOWN – стрелка вниз

-  **Способ кодирования**: ASCII/Unicode коды символов, преобразованные в внутреннее представление Pygame

### 4\.3 Характер и организация выходных данных

**Выходные данные** состоят из двух компонентов:

1. **Визуальное отображение**: Графический вывод в окно приложения размером 1268 × 850 пиксел

   -  Фоновое изображение (неоновый стиль)

   -  Позиции мяча и ракеток

   -  Мины на игровом поле

   -  Счет игроков

2. **Системные события**: Завершение программы при закрытии окна или отправке сигнала выхода

### 4\.4 Формат, описание и способ кодирования выходных данных

-  **Формат вывода**: 2D-растровая графика, отрисовываемая в окно Pygame

-  **Цветовая схема**: RGB (красный, зеленый, синий) каналы, каждый от 0 до 255

-  **Разрешение**: 1268 × 850 пиксел при частоте обновления 120 кадров в секунду

-  **Объекты на экране**:

   -  Фон: PNG-изображение неонового стиля

   -  Мяч: PNG-изображение зеленого мяча (50 × 50 пикс)

   -  Ракетки: PNG-изображения красной и синей ракеток (20 × 200 пикс)

   -  Мины: PNG-изображения красных мин (30 × 30 пикс)

   -  Текст счета: 80pt шрифт, синий цвет (44, 174, 247) для первого игрока, красный (191, 19, 21) для второго

-  **Координаты текста счета**:

   -  Первый игрок: (425, 150)

   -  Второй игрок: (1020, 150)
