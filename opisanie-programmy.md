## 1\. Общие сведения

### 1\.1 Обозначение и наименование программы

**Neo Ping-Pong** -- игра на Python, которая реализует классический пинг-понг с добавлением мин, влияющих на игровой процесс.

### 1\.2 Программное обеспечение, необходимое для функционирования программы

Для работы программы требуется Python версии 3.7 и выше, а также библиотека Pygame для отрисовки графики и обработки событий.

### 1\.3 Языки программирования, на которых написана программа

Программа написана на языке Python с использованием библиотеки Pygame.

## 2\. Функциональное назначение

Neo Ping-Pong реализует классическую игру пинг-понг для двух игроков на одном экране. Основные функции включают управление двумя ракетками, движение и отскоки мяча, система миниатюрных взрывающихся объектов, которые появляются после каждого контакта ракетки с мячом и увеличивают его скорость. Программа предназначена для учебных целей в качестве примера разработки простой игры на Python.

## 3\. Описание логической структуры

### 3\.1 Алгоритм программы

Программа работает на основе циклического обновления состояния игры с частотой 120 кадров в секунду. Каждый кадр обрабатывает события клавиатуры для управления ракетками, обновляет позиции игровых объектов, проверяет столкновения между мячом, ракетками и минами, отслеживает пересечение мячом границ экрана для подсчёта очков, а затем отрисовывает все объекты на экран. При достижении мячом границы экрана счёт противника увеличивается и мяч возвращается в центр.

### 3\.2 Используемые методы

Программа построена на основе объектно-ориентированного программирования с использованием классов для представления игровых объектов. Обнаружение столкновений реализовано через проверку пересечения прямоугольников. Движение объектов осуществляется путём изменения координат на основе векторов скорости. Для создания вариативности используются функции модуля random для выбора направления мяча и размещения мин на игровом поле.

### 3\.3 Структура программы с описанием функций составных частей и связи между ними

Программа состоит из четырёх классов. Класс **Screen** управляет графическим окном, отрисовывает фон и выводит счёт игроков. Класс **Ball** контролирует логику мяча: движение, отскоки от стен и препятствий, проверку столкновений с ракетками и минами. Класс **Player** представляет ракетку с функциями управления движением и хранением счёта. Класс **Mines** описывает взрывающиеся объекты, которые генерируются на случайных позициях. Главный модуль main.py инициализирует объекты и управляет основным игровым циклом, координируя взаимодействие всех компонентов.

### 3\.4 Связи программы с другими программами

Программа зависит от библиотеки Pygame. Вспомогательный модуль settings.py содержит константы конфигурации, которые могут быть изменены без модификации основного кода.

## 4\. Используемые технические средства

Программа может работать на персональных компьютерах под управлением Windows с процессором x86/x64. Требуемый объём оперативной памяти минимален -- около 512 МБ. Минимальное разрешение экрана 1268×850 пикселей. Необходим установленный интерпретатор Python и библиотека Pygame.

## 5\. Вызов и загрузка

### 5\.1 Способ вызова программы с соответствующего носителя данных

Программа запускается командой `python main.py` из директории проекта в командной строке после установки зависимостей.

### 5\.2 Входные точки в программу

Основная входная точка находится в файле main.py, где инициализируются все игровые объекты и запускается основной цикл обработки событий. Вспомогательные входные точки представлены конструкторами классов Screen, Ball, Player и Mines.

## 6\. Входные данные

### 6\.1 Характер, организация и предварительная подготовка входных данных

Входными данными являются события нажатия клавиш от игроков. Первый игрок управляет ракеткой клавишами W и S, второй игрок использует клавиши со стрелками вверх и вниз. Входные данные обрабатываются в реальном времени из потока событий Pygame, предварительная подготовка не требуется.

### 6\.2 Формат, описание и способ кодирования входных данных

События клавиатуры представлены объектами pygame.event с типами KEYDOWN и KEYUP. Каждой клавише соответствует стандартный код Pygame: K_w, K_s, K_UP, K_DOWN.

## 7\. Выходные данные

### 7\.1 Характер и организация выходных данных

Выходные данные представляют собой визуальные элементы, выводимые на экран: фоновое изображение, две ракетки, мяч, мины и текстовое отображение счёта каждого игрока.

### 7\.2 Формат, описание и способ кодирования выходных данных

Визуальные элементы загружаются из PNG-файлов папки assets (фон, ракетки, мяч, мины). Счёт отображается текстом размером 80 пикселей: голубой цвет (RGB 44, 174, 247) для первого игрока и красный (RGB 191, 19, 21) для второго. Отрисовка выполняется через с применением двойной буферизации.
